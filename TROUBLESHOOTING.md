# دليل حل مشاكل الاتصال بالخادم

## المشكلة: "لا يمكن الوصول إلى https://www.crystal4card.com/api"

إذا ظهرت هذه الرسالة عند محاولة الدخول، اتبع الخطوات التالية:

## ✅ الحلول المطبقة

### 1. إضافة Network Security Config
تم إضافة ملف `network_security_config.xml` للسماح بالاتصال بالموقع.

### 2. إضافة أذونات الإنترنت
تم إضافة الأذونات التالية في `AndroidManifest.xml`:
- `INTERNET`: للوصول إلى الإنترنت
- `ACCESS_NETWORK_STATE`: للتحقق من حالة الاتصال

### 3. تحسين معالجة الأخطاء
تم تحسين رسائل الأخطاء لتوضيح المشكلة بشكل أفضل.

## 🔍 خطوات التحقق

### 1. التحقق من الموقع
افتح المتصفح على الهاتف وجرب:
```
https://www.crystal4card.com/api/categories/
```

إذا لم يعمل في المتصفح، المشكلة في الموقع نفسه وليس التطبيق.

### 2. التحقق من الاتصال بالإنترنت
- تأكد من تفعيل Wi-Fi أو بيانات الجوال
- جرب فتح موقع آخر للتأكد من الاتصال

### 3. التحقق من DNS
إذا كان الموقع لا يعمل، قد تكون المشكلة في DNS. جرب:
- تبديل شبكة Wi-Fi
- استخدام بيانات الجوال بدلاً من Wi-Fi

### 4. التحقق من SSL Certificate
إذا كان الموقع يستخدم HTTPS، تأكد من:
- أن شهادة SSL صحيحة
- أن الموقع متاح عبر HTTPS

## 🛠️ حلول إضافية

### إذا استمرت المشكلة:

#### 1. إعادة بناء التطبيق
```bash
cd "abuhussein flutter/my_app"
flutter clean
flutter pub get
flutter build apk --release
```

#### 2. التحقق من إعدادات الاستضافة
- تأكد أن الموقع متاح من الإنترنت (ليس فقط محلياً)
- تحقق من إعدادات Firewall
- تأكد أن الموقع يعمل على المنفذ 443 (HTTPS)

#### 3. اختبار الاتصال من الكمبيوتر
افتح Terminal/Command Prompt وجرب:
```bash
curl https://www.crystal4card.com/api/categories/
```

إذا لم يعمل، المشكلة في الموقع وليس التطبيق.

#### 4. تغيير الرابط مؤقتاً للاختبار
إذا كان لديك رابط آخر للاختبار، يمكنك تغييره في:
`lib/config/api_config.dart`

```dart
static const String baseUrl = "https://your-test-domain.com/api";
```

## 📱 اختبار على الهاتف

1. **افتح التطبيق**
2. **افتح Developer Options** (إذا كان متاحاً)
3. **تحقق من Logcat** لرؤية الأخطاء التفصيلية:
   ```bash
   adb logcat | grep -i "dio\|network\|ssl"
   ```

## ⚠️ ملاحظات مهمة

1. **المحاكي vs الهاتف الحقيقي:**
   - المحاكي يستخدم `10.0.2.2` للوصول إلى localhost
   - الهاتف الحقيقي يحتاج رابط عام متاح من الإنترنت

2. **HTTPS vs HTTP:**
   - Android 9+ يمنع HTTP افتراضياً
   - تأكد من استخدام HTTPS

3. **شهادة SSL:**
   - إذا كانت الشهادة غير موثوقة، قد يرفض Android الاتصال
   - تأكد من أن شهادة SSL صحيحة

## 📞 إذا لم تحل المشكلة

1. تحقق من Logcat للأخطاء التفصيلية
2. تأكد من أن الموقع يعمل من المتصفح
3. جرب تغيير الرابط إلى رابط آخر للاختبار
4. تحقق من إعدادات الاستضافة و Firewall
